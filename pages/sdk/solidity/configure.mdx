# Configure

## Set Up Your Project

### Option A: New Project

Initialize a new project with Tempo support:

```bash
forge init -n tempo my-project && cd my-project
```

This creates a new directory `my-project` with a basic project structure configured for Tempo.

```text
.
├── foundry.lock
├── foundry.toml
├── lib
│   ├── forge-std
│   └── tempo-std
├── README.md
├── script
│   └── Mail.s.sol
├── src
│   └── Mail.sol
└── test
    └── Mail.t.sol
```

### Option B: Existing Project

Add Foundry to an existing project:

```toml [foundry.toml]
[profile.default]
src = "src"
out = "out"
libs = ["lib"]
```

Install Tempo standard library:

```bash
forge install tempoxyz/tempo-std
```

## Configure Foundry for Tempo

:::info
It is currently not possible to use `forge test` or `forge script` with the `--rpc-url` (alias: `--fork-url`) flag due to a known limitation in Foundry.

Instead use `vm.createSelectFork(vm.envString("TEMPO_RPC_URL"))` or `vm.createSelectFork("tempo_testnet")` within your tests or scripts.
:::

Set up the environment variable:

```bash
export TEMPO_RPC_URL=https://rpc.testnet.tempo.xyz
```

Or add to the `foundry.toml` in your project:

```toml [foundry.toml]
[rpc_endpoints]
tempo_testnet = "https://rpc.testnet.tempo.xyz"
```
