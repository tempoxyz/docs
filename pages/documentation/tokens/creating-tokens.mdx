# Creating Tokens

Creating a TIP-20 token on Tempo is achieved by calling the TIP-20 factory contract.

The process deploys a new token with compliance features, role-based permissions, and integration with Tempo's payment infrastructure.

## Example

Create a new token by calling the `createToken` function on the `TIP20Factory` contract.

```solidity 
TIP20Factory factory = TIP20Factory(0x20Fc000000000000000000000000000000000000);

TIP20 token = factory.createToken(
  "My Company USD",  // name
  "MCUSD",           // symbol
  "USD",             // currency
  TIP20(0x20c0000000000000000000000000000000000000), // quoteToken (linkingUSD)
  msg.sender         // admin
);
```

:::note
See a full [TypeScript Example](/sdk/typescript/api/token.create)
:::

When you create a TIP-20 token, you interact with the `TIP20Factory` precompiled contract deployed at a fixed address. The factory:

1. **Validates your parameters** (currency code, quote token, etc.)
2. **Deploys a new TIP-20 contract** at a deterministic vanity address
3. **Grants admin role** to your specified address
4. **Sets up default states** (transfer policy, supply cap, pause state)
5. **Emits a `TokenCreated` event** with the new token's address and ID

## Interface

```solidity 
function createToken(
  /** Name of the token */
  string memory name,
  /** Symbol of the token */
  string memory symbol,
  /** Currency of the token */
  string memory currency,
  /** Linking token */
  TIP20 quoteToken,
  /** Admin of the token */
  address admin
) external returns (
  /** Address of the newly created token */
  address
);
```

## FAQ

### Vanity Addresses

Each TIP-20 token is deployed to a predictable address following this pattern:

```
0x20C0000000000000000000000000<tokenId>
```

The token ID is an incrementing counter managed by the factory, making token addresses easily identifiable as TIP-20 tokens. For example, token ID `1` deploys to `0x20C0000000000000000000000000000001`.

### Quote Tokens

Every TIP-20 token has a **quote token** that determines:

- Which token it can be traded against on the built-in DEX
- Its position in the token hierarchy (used for fee preferences and liquidity routing)

Most USD stablecoins use [linkingUSD](/documentation/tokens/linkingUSD) at `0x20c0000000000000000000000000000000000000` as their quote token.

### Currency Codes

The `currency` parameter automatically sets decimal precision via the [TIP-4217 registry](/protocol/tokens/tip-4217):

- **USD, EUR, GBP**: 6 decimals (representing cents, e.g., `1000000` = $1.00)
- **JPY, KRW**: 0 decimals (whole units only)
- **BTC**: 8 decimals
- **Custom currencies**: Configurable by the TIP-4217 admin

### Quote Tokens

The `quoteToken` parameter determines trading pairs and token hierarchy:

- **Most stablecoins**: Use [linkingUSD](/documentation/tokens/linkingUSD) (`0x20c0000000000000000000000000000000000000`) as their quote token
- **Custom pairs**: Can use any existing TIP-20 token as a quote token
- **linkingUSD**: The root quote token that serves as the base for USD stablecoins

:::warning
Quote tokens cannot be changed after creation. Choose carefully based on your expected liquidity and trading needs.
:::

### Roles & Permissions

The **admin role** is granted to the address you specified. 
It can grant/revoke other roles, change transfer policies, and set supply caps.

:::warning
Note: You will need to [grant roles](/documentation/tokens/roles) for minting, burning, etc, including the admin provided to the `createToken` function.
:::

### Default State

| Setting | Default Value | Description |
|---------|---------------|-------------|
| **Transfer Policy** | `1` (always-allow) | Anyone can send/receive tokens |
| **Supply Cap** | Unlimited | No maximum token supply |
| **Total Supply** | `0` | No tokens exist yet - must be minted |
| **Paused** | `false` | Transfers are enabled |
