# Sending Transactions

Tempo makes EIP-7702 a native transaction type, which simplifies how account abstraction works at the protocol level. Instead of relying on middleware layers, special entry points, or custom relayers, the core logic lives directly in the base layer.

This design choice prioritizes simplicity and consistency:

- **Transactions are easier to parse and verify** — nodes natively understand multi-call transactions, alternative fee tokens, and flexible signatures
- **Wallets integrate directly** — no need for special support contracts or wrapper logic
- **Predictable infrastructure** — fee markets, mempools, and RPCs work without special cases or custom opcodes
- **Future-proof architecture** — the chain supports batching, sponsored gas, and programmable payments without requiring constant protocol upgrades

The result is a cleaner, more predictable foundation for building wallets and applications with powerful transaction capabilities built in.
